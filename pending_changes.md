# 템플릿 개선 대기 항목

*최종 수정: 2026-05-01*

`spec_template_base.html`에 반영되어야 하는 변경 사항 목록. 처리되면 해당 항목 삭제.

---

## 1. 작성일 표시 필요

- **현재**: `version.timestamp`만 데이터에 존재, 사용자 가시 영역엔 노출 없음
- **변경**: 기획서에 작성일(또는 수정일)을 화면상에 표시
- **미정**: 표시 위치 (상단 헤더? 하단 푸터? 패널 내부?)

## 2. 저장 방식 — 템플릿 본체와 카탈로그 모델 정렬

- **현재**: `spec_template_base.html`은 여전히 **단일 파일 + versions[] 누적** 모델. 한편 `sample/reading_log/` 는 **파일 분리 + 정적 인덱스 카탈로그** 모델로 가기로 결정 (vercel 공유 목적).
- **불일치 지점**:
  - 템플릿이 단일 파일 모델이라 `📌 버전 저장`은 versions[]에 새 항목 누적하는 방식 그대로.
  - 카탈로그 운영 흐름은 "버전 저장 → 다운로드 HTML을 versions/에 별도 파일로 추가 + 인덱스에 카드 추가". 즉 다운로드된 HTML이 새 위치에 들어가도록 사람이 옮기는 절차가 있음.
- **미정**:
  - 템플릿의 `📌 버전 저장`을 분리 모델 기준으로 다시 설계할지 (예: 다운로드 파일명을 `v0.X.html`로 통일, versions[]는 자기 버전만 남기게).
  - 인덱스 카드 갱신을 자동화할 수 있는지 (현재는 수동).
- **다음 대화에서**: 대표님과 어디까지 자동화할지 합의 후 템플릿 본체 수정.
