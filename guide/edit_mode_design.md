# 화면 설계 템플릿 — 편집 모드 설계

*작성일: 2026-04-18*

## 배경

템플릿에는 두 개의 편집 경로가 존재함:

1. **Claude Code에게 요청** → HTML 파일 직접 수정
2. **브라우저 편집 모드에서 수정** → localStorage 경유

두 경로 간 동기화, 버전 관리, 변경 추적이 필요함. 본 문서는 이를 위한 설계를 정리한 것.

---

## 1. 저장 모델

### 1.1 Single Source of Truth (SoT)

**HTML 파일 = 정본.** localStorage는 임시 포스트잇.

**비유:**
- HTML 파일 = 공식 문서
- localStorage = 책상 위 포스트잇 (내 브라우저에만 있음)
- 진짜 반영하려면 도장 찍어서 공식 문서에 등록해야 함

### 1.2 3단계 저장

| 단계 | 비유 | 저장 위치 | 신뢰도 | 용도 |
|---|---|---|---|---|
| 자동 저장 | 포스트잇 | localStorage | ⚠️ 임시 | 편집 도중 사고 방지 |
| 💾 **작업본 저장** | 서랍 | HTML `draft` 슬롯 | ✅ 파일 저장 | 이어서 작업, 공유 |
| 📌 **버전 저장** | 공식 도장 | HTML `VERSIONS[]` | ✅ 영구 | 마일스톤 |

### 1.3 localStorage 주의사항

localStorage는 다음 상황에 **사라질 수 있음**:

- 브라우저 데이터 초기화
- 시크릿/프라이빗 모드
- 다른 브라우저/PC에서 열기
- HTML 파일 경로·이름 변경
- 브라우저 재설치

→ 중요한 내용은 **반드시 💾 또는 📌**로 저장.

## 2. 버전 체계

### 2.1 번호 체계 — 메이저/마이너

- 첫 버전: `v0.1`
- 마이너 → 뒷자리 +1 (`v0.2` → `v0.3`)
- 메이저 → 앞자리 +1, 뒷자리 0 (`v0.5` → `v1.0`)

**판단 기준 (가이드라인)**

| 메이저 올리는 경우 | 마이너로 두는 경우 |
|---|---|
| 공식 리뷰 통과 | 문구·디테일 수정 |
| 방향성 재설계 | 섹션 추가/삭제 |
| 외부 공유·최종 버전 | 피드백 반영 |

### 2.2 버전 메타데이터

각 버전에 저장되는 정보:

```js
{
  id: 'v0.3',
  timestamp: '2026-04-18T14:30:00Z',
  author: 'claude' | 'manual',      // 🤖 Claude / ✏️ 편집모드
  memo: '피드백 반영',                // 선택사항 (비우면 "메모 없음")
  specs: { ... },                    // 이 버전의 핀 상태 전체 스냅샷
  changes: [                         // 이 버전에서 바뀐 것 목록
    { pin: 3, field: '역할', action: '수정' },
    { pin: 4, action: '추가' }
  ]
}
```

**원칙:**
- **메모**: 선택사항
- **Author**: `🤖 Claude` / `✏️ 편집모드` 구분 표시
- **Timestamp**: **버전 생성 시점**만 기록 (필드별 편집 시각은 기록하지 않음)

## 3. 변경 기록

### 3.1 Diff 스코프 — 핀 + 필드 레벨

**① 핀 레벨**
- ✨ 추가: 초록 테두리
- 🗑 삭제: 회색 고스트 (원래 자리에 반투명)
- ✏️ 수정: 주황 테두리

**② 필드 레벨**
- 패널 열면 바뀐 섹션(역할/상태/속성/Confluence)에 **노란 배경**
- *"역할 · 상태 2개 변경"* 같은 요약 배지

*값 레벨(구체 텍스트 diff)은 초기 범위 밖. 필요 시 향후 추가.*

### 3.2 기록 구조

버전 안 `changes` 배열에 한 줄씩 기록 (위 2.2 예시 참조).

## 4. UI / UX

### 4.1 상단 바

- **버전 드롭다운** (`v0.3 ▾`)
  - 예시 항목:
    ```
    v0.3  오늘 14:30 · 🤖 Claude
    v0.2  어제 · ✏️ 편집모드 — 피드백 반영
    v0.1  3일 전 · ✏️ 편집모드
    ```
- **🔀 이전과 비교** 토글
- **저장 상태 표시**
  | 상태 | 표시 |
  |---|---|
  | 저장됨 | `● 저장됨` (회색) |
  | 수정 중 | `● 미저장` (주황) |
  | 💾 직후 | `💾 작업본 저장됨 · HH:MM` |
  | 📌 직후 | `📌 v0.3 저장됨` |

### 4.2 이탈 경고

미저장 상태에서 **탭 닫기 / 새로고침 / 다른 URL 이동** 시 브라우저 기본 경고창 표시.

*(브라우저 제약: 경고 메시지 텍스트 커스터마이징 불가)*

### 4.3 비교 모드

- **비교 대상**: 선택된 버전의 **직전 버전** (자동)
- **편집 중 동작**: 내 draft vs 최신 버전 — **저장 전 미리보기** 용도
- 비교 모드 진입 시 상단에 요약 띠: *"v0.2 → v0.3: 핀 2개 수정, 1개 추가"*

## 5. 충돌 처리

### 5.1 시나리오

PM이 편집 중인 동안 Claude가 HTML을 수정하면 → base 버전 불일치 발생.

```
월요일 10시  PM: HTML 열고 편집 시작 (draft, base=v0.3)
월요일 15시  Claude: HTML을 v0.4로 업데이트
월요일 18시  PM: 브라우저로 돌아옴
             → 내 draft는 v0.3 기준, 공식 문서는 v0.4
             → 충돌 감지
```

### 5.2 동작

파일 열 때 draft의 base 버전 ↔ HTML 최신 버전 비교.
다르면 모달:

```
⚠️ 그 사이 문서가 바뀌었어요
  내 편집 기준 버전: v0.3
  공식 문서 최신 버전: v0.4

  [내 편집 버리기]         → v0.4 상태로 시작
  [내 편집을 새 버전으로]   → v0.4 위에 내 편집 얹어 v0.5로 커밋
```

**자동 머지 없음** — 사용자가 명시적으로 선택.

## 6. 남은 결정 (구현 중 확정, 기본안 적용)

| # | 항목 | 기본안 |
|---|---|---|
| 1 | 📌 버전 저장 모달 | 메이저/마이너 라디오 + 메모 입력 + "저장 & 다운로드" |
| 2 | 💾 작업본 저장 | 버튼 클릭 시 바로 HTML 다운로드 (확인 모달 없음) |

| 4 | draft 복원 | 파일 열 때 draft 있으면 *"이어서 하시겠어요?"* 제안 |
| 5 | 비교 모드 시각 | 핀 색(초/주/회) + 바뀐 섹션 노란 배경 + 요약 배지 |
